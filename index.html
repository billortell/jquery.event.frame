What's this, then?

Provides an animation timer that fires frame events at bound elements. The timer starts the instant a frame event is bound (to any element), and stops after all events are unbound, so there is only ever one timer. Frame events are jQuery internal only, that is, they are not bound as browser native events.

Besides the usual event properties, a frame event object has 'frameCount'. e.frameCount is reset to 0 whenever the timer is restarted.

.bind('frame') | .unbind('frame')
.bind('frame') | .unbind('frame')

You may have noticed that each .bind launches the timer at a different speed. Speed can be set by passing in a data object with the property frameDuration: Int. The default is 40 (25 frames/second). I've made this demo go a lot slower than that so you can see what is going on.

Reporting problems

github.com/stephband/jquery.event.frame/issues

Please note that jquery.event.frame works only in jQuery 1.4 or higher. Until that is released, the download includes a version of jQuery nightly. Do not use this version of jQuery in production.
